Clodex node "{{ .Release.Name }}" deployed to namespace {{ .Release.Namespace }}.
{{- if not .Values.secrets.wireToken }}

Before the pod goes Ready, the secret must exist (if you haven't yet;
tr -d '\n' matters — a trailing newline corrupts the bearer token):

  kubectl -n {{ .Release.Namespace }} create secret generic {{ include "clodex.secretName" . }} \
    --from-literal=wire-token="$(openssl rand -hex 24)" \
    --from-literal=oauth-token="$(tr -d '\n' < TOKEN-FILE)"

(omit oauth-token when the model credential comes from the pod's cloud
identity instead — e.g. Bedrock via an IRSA-bound service account.)
{{- end }}

Add the node to clodexctl (resolves the pod through YOUR kubectl access —
no ingress exists):

  clodexctl ctx add {{ .Release.Name }} \
    --kubectl svc/{{ include "clodex.name" . }} \
    --namespace {{ .Release.Namespace }} \
    --kube-context $(kubectl config current-context) \
    --token "$(kubectl -n {{ .Release.Namespace }} get secret {{ include "clodex.secretName" . }} -o jsonpath='{.data.wire-token}' | base64 -d)"

Then: clodexctl --ctx {{ .Release.Name }} spawn worker --type claude
      clodexctl --ctx {{ .Release.Name }} attach worker
{{- if .Values.web.enabled }}
      clodexctl --ctx {{ .Release.Name }} web        # browser GUI (port {{ .Values.web.port }})
{{- end }}
